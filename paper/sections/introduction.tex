\section{Introduction}

Dense retrieval systems have become the dominant paradigm for information retrieval, powering applications ranging from general web search to retrieval-augmented generation. A critical component of these systems is document chunking: the process of segmenting long documents into smaller, retrievable units. The chunking strategy directly impacts retrieval effectiveness: chunks that are too small may lack sufficient context, while chunks that are too large may dilute semantic signals and exceed embedding model capacity.

The traditional approach segments documents first and then encodes each chunk independently using a dense encoder. This \textit{pre-embedding chunking} paradigm has been the foundation for most retrieval systems, with variations primarily differing in how segmentation boundaries are determined: from simple fixed-size windows to sophisticated LLM-guided approaches~\cite{densex2024,lumberchunker2024}. Recently, \textit{contextualized chunking} methods have emerged, enabled by modern embedding models with extended context windows that support thousands of tokens beyond BERT's 512-token limit~\cite{e5,jina,bge}. Methods like Late Chunking~\cite{latechunking2024} exemplify this paradigm shift: they first encode the entire document to obtain contextual embeddings, then perform segmentation while preserving cross-chunk contextual information.


Simultaneously, segmentation methods themselves have evolved considerably. Beyond traditional fixed-size and sentence/paragraph-based approaches~\cite{para}, recent work has introduced semantic boundary-based detection~\cite{propositions2024} and LLM-guided chunking~\cite{densex2024,lumberchunker2024}. However, these diverse methods are evaluated inconsistently across different retrieval scenarios, making systematic comparison challenging. For instance, proposition-based methods were evaluated using constrained-input models (typically 512-token limitation) on QA datasets~\cite{propositions2024}, while LumberChunker was evaluated on a fundamentally different task—single-book paragraph retrieval where each query is pre-scoped to a specific book—using commercial embedding APIs~\cite{lumberchunker2024}, and Late Chunking was evaluated on selected BEIR passage retrieval datasets for multi-document retrieval~\cite{latechunking2024}. This heterogeneity in evaluation settings span not only different embedding models, context window sizes, and datasets, but also fundamentally different retrieval tasks (single-document vs. multi-document retrieval), makes it impossible to determine whether performance differences stem from the chunking strategies themselves or from confounding factors.


To address these gaps, we investigate the following research questions:

\textbf{RQ1} \textit{How do different segmentation methods compare in standard multi-document retrieval?} We evaluate fixed-size, sentence/paragraph-based, semantic boundary-based, and LLM-guided chunking methods (Propositional-based and Lumbar chunker) using traditional pre-embedding chunking on standard BEIR benchmarks. 

\textbf{RQ2} \textit{Do segmentation methods generalize across retrieval paradigms?} We apply the same segmentation methods to single-document needle-in-a-haystack retrieval, where each query targets a specific long document. Using GutenQA dataset from LumbarChunker, we test whether the relative strengths observed in multi-document settings hold when the retrieval scope is constrained to individual documents, revealing which methods are task-specific versus universally effective.

\textbf{RQ3} \textit{Can contextualized chunking provide consistent improvements across different configurations?} We apply contextualized chunking to all segmentation methods from RQ1 across both retrieval paradigms from RQ2. We compare pre-embedding versus contextualized versions of each method to determine whether contextualized chunking provides universal benefits or only enhances specific segmentation strategies.

\textbf{RQ4} \textit{What factors determine when contextualized chunking outperforms pre-embedding approaches?} We implement Late Chunking as a representative contextualized method and systematically vary chunk size (128, 256, 512, 1024 tokens), overlap ratios (0\%, 25\%, 50\%), and analyze performance across different document types (narrative texts vs. reference documents) and query characteristics (factual vs. conceptual). We measure both retrieval effectiveness and computational overhead (latency, memory usage) to identify when contextualized chunking justifies its additional cost.
