\section{Experimental Setup}

We evaluate each chunking strategy using the unified framework described in Section~\ref{sec:benchmarking_document_segmentation_strategies}. This section details the specific implementations, datasets, and evaluation procedures used in our experiments.

\subsection{Segmentation Methods}
To ensure a fair comparison, we use a consistent implementation for each segmentation strategy across all datasets.

For \textbf{structure-based heuristics}: the \textbf{Paragraphs} strategy defines chunks by newline characters, collapsing consecutive blank lines into a single boundary. The \textbf{Fixed-Size} method divides documents into uniform 512-token segments with no overlap. Finally, the \textbf{Sentences} approach creates chunks from single sentences, as identified by the spaCy `en\_core\_web\_sm` model.

For \textbf{semantically-informed and LLM-guided strategies}, the \textbf{Semantic Sentence} strategy follows the open-source implementation from~\cite{semantic-splitting-source}, using \texttt{jina-embeddings-v2-small-en} to compute the cosine similarity between adjacent sentences and inserting a boundary when the similarity falls below 0.5. For both \textbf{Proposition-Based} and \textbf{LumberChunker} strategies, we prompt the \texttt{Gemini-1.5-Flash} model for chunking document, following previous work from~\cite{duarte_lumberchunker_2024}, to make sure output as deterministic as possible, we set the model generation temperature to 0.0.

\subsection{Embedding Models Used}

For embedding models used to evaluate the effectiveness of different segmantation strategies and embedding paradigm, we select the following four models:

\begin{itemize}
	\item \textbf{jina-embeddings-v2-small-en}
	
	\item \textbf{jina-embeddings-v3}
	
	\item \textbf{nomic-embed-text-v1}
	
	\item \textbf{multilingual-e5-base}
\end{itemize}


While the first three models were evaluated in the initial late-chunking method, we also select an additional model multilingual-e5-base to test its robustness, as a nature of late-chunking that has to use average pooling for representation, our selection is constrained by selecting from Mteb leaderboard the highest performing embedding model that is using mean pooling for text representation.

While all of the embedding models are able to handle the \textit{pre-embedding} cases, where each document is firstly chunked and encoded seperatly; during ~\textit{Contextualized chunking}, it requires to input multiple chunks together to the embedding model, therefore a maximum possible chunk approach is used, that is: each time it will feed the maximum possible number of chunks in a document into the context window of the embedding model; once it exceed, it will create another chunking.

\subsection{Datasets and Evaluation}
Our evaluation is designed to test the generalizability of chunking strategies across different retrieval tasks and domains.

\textbf{In-Document Retrieval} is evaluated using the \textbf{GutenQA} dataset~\cite{duarte_lumberchunker_2024}, which contains over 50 classic literature books. For each book, a set of questions asks for specific information contained within that book. This task tests for precise information localization within a single, long document.

\textbf{In-Corpus Retrieval} is evaluated on a selection of datasets from the \textbf{BEIR} benchmark~\cite{thakur_beir_2021}: FiQA, ArguAna, SciDocs, TREC-COVID, SciFact, and NFCorpus. We chose these specific datasets based on two criteria. First, they cover a diverse range of retrieval tasks and domains, from finance and scientific articles to biomedical texts. Second, these datasets have a relatively small number of documents, which was a practical consideration to manage the computational cost associated with the LLM-guided chunking methods.

All text is lowercased and tokenized using the specific tokenizer provided with each embedding model.

For evaluation metrics, we use 
Following standard practices for these datasets, we report nDCG@10 (normalized Discounted Cumulative Gain at rank 10) for all BEIR tasks and DCG@10 for GutenQA. The formula for DCG is:
$$ \mathrm{DCG@10} = \sum_{i=1}^{10} \frac{2^{\mathrm{rel}_i} - 1}{\log_2(i+1)} $$
where $\mathrm{rel}_i$ is the relevance score of the result at rank $i$. The nDCG@10 metric normalizes this score by the best possible DCG@10 score, producing a value between 0 and 1. All metrics are calculated using the official BEIR evaluation toolkit.
