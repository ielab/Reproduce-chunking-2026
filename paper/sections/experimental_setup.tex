\section{Experimental Setup}

We evaluate each chunking strategy using the unified framework described in Section~\ref{sec:benchmarking_document_segmentation_strategies} and illustrated in Figure~\ref{fig:chunking-embedding-pipeline}. This section details the specific implementations, datasets, and evaluation procedures used in our experiments.

\subsection{Segmentation Methods}
We used a consistent implementation for each segmentation strategy across all datasets to ensure comparability.

\subsubsection{Structure-based Heuristics}

\begin{itemize}
	\item \textbf{Paragraphs:} Chunks are defined by newline characters. Consecutive blank lines are collapsed into a single boundary.
	\item \textbf{Fixed-Size:} Documents are divided into 512-token segments with no overlap.
	\item \textbf{Sentences:} We group five consecutive sentences, as identified by the spaCy `en\_core\_web\_sm` model, into each chunk.
\end{itemize}

\subsubsection{Semantically Informed and LLM-guided Strategies}

\begin{itemize}
	\item \textbf{Semantic Sentence:} Following the open-source implementation from~\cite{semantic-splitting-source}, we use `jina-embeddings-v2-small-en` to compute the cosine similarity between adjacent sentences and insert a boundary when the similarity falls below 0.5.
\item \textbf{LLM-Guided (LumberChunker):} We prompt `Gemini-1.5-Flash'  for the GutenQA dataset and `GPT-4.1-Nano` for the BEIR datasets, following the settings in~\cite{duarte_lumberchunker_2024}.
\item \textbf{Proposition-Based:} For the GutenQA dataset, we first apply LumberChunker and then use a second `GPT-4.1-Nano' prompt to break the resulting chunks into propositions. For BEIR, we generate propositions directly from the text. For all LLM-based methods, we set the temperature to 0.0 for deterministic output.
\end{itemize}


\subsection{Embedding model and Paradigms}


We test four different embedding models: `jina-embeddings-v2-small-en`, `jina-embeddings-v3`, `nomic-embed-text-v1`, and `multilingual-e5-base`. Each model is used in both the pre-embedding and contextualized paradigms.
\begin{itemize}
	\item In the \textbf{pre-embedding paradigm}, each chunk is encoded independently.
	\item In the \textbf{contextualized paradigm}, the full document is encoded to obtain token-level embeddings. The final vector for each chunk is then calculated by taking the mean of the embeddings of the tokens it contains (mean pooling).
\end{itemize}
For very long documents in the contextualized paradigm, we use a sliding window approach where the document is split into overlapping sections that fit the model's context window~\cite{gunther_late_2025}.



\subsection{Datasets and Evaluation}

\subsubsection{Datasets}
We evaluate the chunking strategies on two different types of retrieval tasks to assess their generalizability.
\begin{itemize}
	\item \textbf{In-Document Retrieval} is evaluated using the \textbf{GutenQA} dataset~\cite{duarte_lumberchunker_2024}, which contains over 50 classic literature books. For each book, there are specific query/questions asking about relevant information in this book. This task tests for precise information localization within a single, long document.
	\item \textbf{In-Corpus Retrieval} is evaluated on a selection of datasets from the \textbf{BEIR} benchmark~\cite{thakur_beir_2021}. We chose these specific datasets: FiQA, ArguAna, SciDocs, TREC-COVID, SciFact, and NFCorpus, based on two criteria. First, they cover a diverse range of retrieval tasks and domains, from finance and scientific articles to biomedical texts. Second, these datasets have a relatively small number of documents, which was a practical consideration to manage the computational cost associated with the LLM-guided chunking methods.
\end{itemize}
All text is lowercased and tokenized using the specific tokenizer provided with each embedding model.

\subsubsection{Evaluation Metrics}
Following standard practices for these datasets, we report nDCG@10 (normalized Discounted Cumulative Gain at rank 10) for all BEIR tasks and DCG@10 for GutenQA. The formula for DCG is:
$$ \mathrm{DCG@10} = \sum_{i=1}^{10} \frac{2^{\mathrm{rel}_i} - 1}{\log_2(i+1)} $$
where $\mathrm{rel}_i$ is the relevance score of the result at rank $i$. The nDCG@10 metric normalizes this score by the best possible DCG@10 score, producing a value between 0 and 1. All metrics are calculated using the official BEIR evaluation toolkit.
